# Agent Instructions

## Project goal

Build a professional personal website for Adam Jele focused on behavioural science, research, psychology, data, decision-making, and AI-adjacent work.

## Visual direction

The site should feel like a dark research lab, editorial portfolio, and high-end technical interface combined.

Desired qualities:

- Sleek
- Dark
- Elegant
- Scientific
- Modern
- Slightly futuristic
- Professional
- Old/new contrast
- Precise rather than loud

Avoid:

- Generic corporate portfolio templates
- Overused neon cyberpunk styling
- Startup cliché visuals
- Heavy animation that harms readability
- Framework bloat
- Random redesigns without a reason

## Current homepage concept

The first screen should be the key visual moment:

- Dark background
- Adam Jele centred on screen
- Name framed by large bracket-like symbols
- Serif name typography for elegance and weight
- Monospace microcopy for scientific/technical atmosphere
- Slowly changing phrase below the name
- Floating keywords around the top-right and bottom-left areas
- Some subtle interference/glitch elements
- Occasional random symbols such as `%$£!^`, binary, Greek letters, and technical fragments

Keywords currently used include:

- Behavioural science
- Neuroscience
- Psychology
- Decision-making
- Data
- Python
- Signal
- Future
- Cyborg-adjacent
- Cats
- Implicit methods
- Quant research
- Human behaviour
- Strategy
- Evidence
- AI workflows

The humour should stay subtle. Cats/cyborg/future/noise are allowed, but the site should still feel employable and premium.

## Technical rules

- Use plain HTML, CSS, and vanilla JavaScript.
- Do not add React, Vue, npm, Tailwind, Bootstrap, or build tooling unless explicitly requested.
- Keep GitHub Pages compatibility.
- Keep the root `index.html` as the entry point.
- Keep CSS in `styles.css` unless the project grows significantly.
- Keep JavaScript minimal and non-essential.
- The page must still work if JavaScript fails.
- Respect `prefers-reduced-motion`.
- Keep responsive behaviour clean on mobile.

## Accessibility rules

- Maintain readable contrast.
- Do not rely only on animation to communicate meaning.
- Keep navigation usable by keyboard.
- Avoid excessive motion.
- Do not hide important text inside images.

## Content rules

Position Adam as a researcher/analyst working across:

- Behavioural science
- Quantitative research
- Psychology
- Decision science
- Data analysis
- AI-assisted workflows
- Human behaviour and strategy

Tone:

- Concise
- Intelligent
- Specific
- Professional
- Not over-explained
- Not buzzword-heavy unless grounded in concrete work

## Development priorities

1. Preserve the first-screen visual concept.
2. Refine homepage sections below the fold.
3. Add selected work/project cards.
4. Add proper contact and CV links.
5. Later: add project detail pages, writing, experiments, dashboards, and case studies.

## Change discipline

When editing:

- Do not rewrite the whole site unnecessarily.
- Preserve existing aesthetic unless asked to change it.
- Make small, deliberate changes.
- Explain what changed briefly.
- Avoid adding dependencies.
