## Format Guide

Use this format for future entries:

```markdown
## [Version] - YYYY-MM-DD

### Added
- New features

### Changed
- Changes to existing functionality

### Fixed
- Bug fixes

### Removed
- Removed features

### Deprecated
- Soon-to-be removed features
```

### Versioning

This project uses [Semantic Versioning](https://semver.org/):
- **MAJOR**: Breaking changes or significant redesigns
- **MINOR**: New features, new pages, new functionality
- **PATCH**: Bug fixes, content updates, small improvements
